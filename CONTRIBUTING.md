# Contributing to Super-Spec

Thank you for your interest in improving Super-Spec.

## What We're Looking For

Super-Spec's core value is its **anti-corner-cutting catalog** — the systematic documentation of how AI models circumvent instructions and the operational definitions that prevent each tactic.

### 1. New Anti-Corner-Cutting Patterns

If you've observed an AI agent shortcutting in a way not yet cataloged, we want to document it. A good pattern report includes:

- **What you said**: The instruction you gave the agent
- **What it did**: How the agent circumvented the instruction
- **Operational definition**: A checkable, non-ambiguous definition that prevents the shortcut
- **Real example**: A concrete before/after from your project

### 2. Platform-Specific Adaptations

Contributions that adapt the skill for specific environments (Cursor rules, Copilot instructions, Windsurf, etc.) are welcome.

### 3. Documentation Improvements

Clearer explanations, better examples, additional diagrams. If something confused you, it will confuse others.

## Contribution Process

1. **Fork** the repository
2. **Create a branch** named `feature/your-feature` or `fix/your-fix`
3. **Make your changes** — keep commits focused and well-described
4. **Open a Pull Request** with a clear description and before/after examples

## Standards

- **SKILL.md** follows the [Agent Skills](https://agentskills.io) standard
- **Documentation** uses American English with standard computer science terminology
- **Commit messages** follow [Conventional Commits](https://www.conventionalcommits.org/)

## Questions?

Open an issue with the `question` label.
