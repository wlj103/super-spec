# 参与贡献 Super-Spec

感谢你对改进 Super-Spec 的兴趣。

## 我们寻找什么

Super-Spec 的核心价值在于其**反偷懒目录**——系统性地记录 AI 模型如何绕过指令，以及阻止每种手段的操作定义。

### 1. 新的反偷懒模式

如果你观察到 AI Agent 以尚未收录的方式偷懒，我们希望记录它。一份好的模式报告包括：

- **你说了什么**：你给 Agent 的指令
- **它做了什么**：Agent 如何绕过指令
- **操作定义**：一个可检查、无歧义的定义，阻止这种偷懒
- **真实示例**：来自你项目的具体前后对比

### 2. 平台适配

欢迎将本 skill 适配到特定环境（Cursor rules、Copilot instructions、Windsurf 等）。

### 3. 文档改进

更清晰的解释、更好的示例、额外的图示。如果某处让你困惑，它也会困惑别人。

## 贡献流程

1. **Fork** 仓库
2. **创建分支**，命名为 `feature/你的功能` 或 `fix/你的修复`
3. **进行更改** — 保持提交聚焦、描述清晰
4. **提交 Pull Request**，附上清晰的描述和前后对比示例

## 标准

- **SKILL.md** 遵循 [Agent Skills](https://agentskills.io) 标准
- **文档** 使用美式英语，配合标准计算机科学术语
- **提交信息** 遵循 [Conventional Commits](https://www.conventionalcommits.org/)
